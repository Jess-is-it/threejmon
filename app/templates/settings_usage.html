Modem Auto Reboot

Modem Auto Reboot depends heavily on TR-069 / GenieACS. A reboot can only be sent when the account is matched to a GenieACS device, and the ONT/modem must support a remote reboot action. If the device has no TR-069 mapping or the ONT rejects reboot commands, the Usage page will record that failure in Rebooted History.
Accounts that repeatedly fail reboot attempts are marked Reboot Blocked. Once an account reaches the maximum failed checks below, the system stops rapid retries and only performs a checker attempt after the configured interval. This prevents accounts with no TR-069 mapping or rejected reboot commands from creating thousands of failed attempts.
Accounts are rebooted as soon as they enter Usage Issues. After a successful reboot, the system waits {{ settings.modem_reboot.buffer_hours }} hour(s) before allowing another reboot for the same account. Failed attempts do not start the buffer; they retry using the retry settings below. Five minutes after a successful reboot, the system checks the account again and clears stale Usage Issues when live traffic has already returned.
Automatically sends a GenieACS reboot task when an account enters Usage Issues.
